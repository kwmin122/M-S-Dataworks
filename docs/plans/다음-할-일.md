## 1. 환경 구성 (P0) — 아무것도 안 되는 상태 탈출

### 1-1. PostgreSQL + Prisma 마이그레이션 실행

```bash
# Docker로 DB 띄우기
docker run -d \
  --name kirabot-pg \
  -e POSTGRES_DB=kirabot \
  -e POSTGRES_USER=kirabot \
  -e POSTGRES_PASSWORD=kirabot \
  -e PGTZ=UTC \
  -p 5432:5432 \
  postgres:16

# .env 설정
# DATABASE_URL="postgresql://kirabot:kirabot@localhost:5432/kirabot?timezone=UTC"

# 마이그레이션 실행
cd web_saas
npx prisma migrate deploy
npx prisma generate
```

완료 기준: `npx prisma studio` 에서 테이블 확인

### 1-2. 환경변수 파일 정리

`web_saas/.env` 항목:
```
DATABASE_URL=...
OPENAI_API_KEY=...
WEBHOOK_SECRET=...        # n8n↔Next.js HMAC 서명용 (임의 문자열)
FASTAPI_URL=http://localhost:8001
NEXT_PUBLIC_APP_URL=http://localhost:3000
```

`rag_engine/.env` 항목:
```
OPENAI_API_KEY=...
```

## 2. 나라장터 API 연결 (P0) — API 키 이미 있음

> **현황**: API 키 확보 완료. n8n 워크플로우 JSON (`n8n/workflows/g2b_bid_crawler.json`) 작성 완료.

### 2-1. 나라장터 Open API 키 설정

- 발급받은 API 키를 n8n 환경변수 `G2B_API_KEY`에 등록
- 엔드포인트: `https://apis.data.go.kr/1230000/BidPublicInfoService04/getBidPblancListInfoServc`

### 2-2. n8n 설치 및 워크플로우 임포트

```bash
# Docker로 n8n 띄우기
docker run -d \
  --name kirabot-n8n \
  -e N8N_BASIC_AUTH_ACTIVE=true \
  -e N8N_BASIC_AUTH_USER=admin \
  -e N8N_BASIC_AUTH_PASSWORD=admin \
  -e WEBHOOK_URL=http://localhost:5678 \
  -p 5678:5678 \
  n8nio/n8n

# 워크플로우 임포트 순서:
# 1. g2b_bid_crawler.json
# 2. ingestion_worker.json
# 3. evaluation_worker.json
# 4. stale_lock_reclaim.json
```

### 2-3. n8n→Next.js 웹훅 연동 테스트

- Next.js 서버 실행 후 `POST /api/webhooks/n8n` 수동 호출
- HMAC 서명 검증 통과 확인

완료 기준: 나라장터 공고 1건이 `bid_notices` 테이블에 INSERT되는 것 확인
