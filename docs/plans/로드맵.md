# Kira Bot 사용자 이용 로드맵 (Phase 8 설계안)

작성일: 2026-02-22
범위: 기존 Phase 1~7(수집/분석/평가 인프라) 이후, 사용자 가치 중심 Phase 8 제품화

---

## 1) 목표 요약

Kira Bot을 단순 공고 검색기가 아니라, 아래 5가지를 한 번에 제공하는 "올라운더"로 포지셔닝한다.

- 공고 수집/검색
- 첨부파일 내부 검색
- GO/NO-GO 의사결정
- 제안서 초안 자동 생성
- 대화형 UX 기반 업무 자동화

포지셔닝 문장:
- `클라이언트형 수집 + 조달 분석 + 전략 판단 + 문서 생성 + 대화형 UX = Kira Bot 올라운더`

주의:
- 경쟁사별 기능은 공개 자료 기준으로만 확인 가능하며, 미확인 영역은 가정으로 분리한다.

## 2) 사용자 이용 플로우 (End-to-End)

### A. 초기 온보딩 (첫 1회)
1. 회사 기본정보 입력
- 업종, 면허, 자본금, 주요 실적, 인증, 활동 지역
2. 관심조건 저장
- 키워드(다중), 지역, 금액 범위, 마감 제외 여부, 기간
3. 알림 채널 연결
- 이메일, 메신저(추후), 대시보드 알림

완료 조건:
- 저장된 회사 프로필 + 관심조건으로 자동 검색/평가가 동작

### B. 일일 운영 플로우 (매일)
1. 검색 시작
- 인터뷰 모드 또는 폼 모드 선택
2. 공고 후보 수집
- 나라장터 기준 필터링 + 첨부파일 텍스트 포함 인덱싱
3. 다중 공고 동시 평가
- N개 공고 GO/NO-GO + 부족조건 + 준비액션 자동 생성
4. 결과 소비
- 대시보드 카드 + 엑셀 다운로드 + 상세 근거 확인
5. 후속 실행
- 관심 공고 선택 → 제안서 초안 생성 워크플로우 진입

### C. 주간/월간 운영 플로우 (관리자)
1. 주간:
- GO 전환율, NO-GO 주요 사유, 반복 탈락 조건 점검
2. 월간:
- 업종/지역별 전략 수정, 키워드 리밸런싱, 템플릿 개선
3. 분기:
- 신규 버티컬 확장 여부(민간입찰/HR/법무/회계) 의사결정

## 3) Module별 사용자 여정 설계 (Phase 8)

### Module 1. 스마트 검색 (인터뷰/폼)
핵심 경험:
- 인터뷰 모드: 챗봇이 질문으로 조건 완성
- 폼 모드: 숙련 사용자가 한 번에 제출

성공 지표:
- 검색 시작 후 첫 결과 도달 시간(TTFR)
- 검색 조건 저장/재사용률

### Module 2. 다중 공고 분석 + 엑셀 출력
핵심 경험:
- 한 번에 N건 평가, 표준 컬럼(공고명/금액/마감/지역/GO/NO-GO/부족조건/URL)

성공 지표:
- 엑셀 내보내기 사용률
- 평가 결과 검토 완료율

### Module 3. 첨부파일 전문 검색
핵심 경험:
- 제목이 아닌 HWP/PDF 본문 기반 검색

성공 지표:
- 첨부문서 기반 검색 적중률
- 본문 키워드로 발견된 신규 공고 비중

### Module 4. 발주계획 사전감지
핵심 경험:
- 공고 게시 전 단계(사전규격/예산 정보) 신호 탐지 및 선제 알림

성공 지표:
- 사전탐지 알림 대비 실제 공고 전환율
- 사전탐지~공고게시 리드타임

### Module 5. 제안서 초안 자동 작성
핵심 경험:
- 회사 템플릿 업로드 후 공고별 초안 자동 채움(HWP/DOCX)

성공 지표:
- 초안 생성 시간 단축률
- 사용자 수정량(수동 편집 비율)

### Module 6. 강점 분석 + 공고 요약 카드
핵심 경험:
- "우리 회사 강점 vs 공고 요구사항"을 1페이지 카드로 즉시 제공

성공 지표:
- 카드 조회 후 액션 실행률
- NO-GO에서 GO로 전환된 재평가율

### Module 7. HWP 분석 & 문서 작성
핵심 경험:
- HWP 업로드 시 자격요건 추출 + 형식 유지 기반 문서 작성

성공 지표:
- HWP 파싱 성공률
- 형식 보존 만족도

## 4) Phase 8 릴리즈 로드맵 (기업형)

### Wave 1 (4주): 사용자 체감 핵심 완성
- Module 1 + Module 2 안정화
- 대시보드/엑셀 출력 표준화
- 인터뷰 모드 질문 품질 개선

출시 게이트:
- 검색→평가→엑셀 다운로드 E2E 95% 이상 성공

### Wave 2 (4~6주): 검색 정확도 차별화
- Module 3 고도화(HWP/PDF 인덱싱)
- Module 6 카드/강점 분석 동시 출시

출시 게이트:
- 첨부파일 본문 검색 Recall 기준 충족

### Wave 3 (6주): 실행 자동화 전환
- Module 5 제안서 초안 생성
- Module 7 HWP 작성/편집 품질 개선

출시 게이트:
- 템플릿 기반 초안 생성 실패율 임계치 이하

### Wave 4 (병행 R&D, 6주+): 선제형 시장 대응
- Module 4 발주계획 사전감지
- 알림 전략(A/B) 및 리드타임 최적화

출시 게이트:
- 사전탐지 알림의 실공고 전환 정합성 검증

## 5) 운영 KPI (기업용)

- Acquisition: 팀 온보딩 완료율, 1주 내 재방문율
- Activation: 첫 검색→첫 GO/NO-GO 도달 시간
- Engagement: 주간 활성 조직 비율, 알림 클릭률
- Value: GO 공고 실제 입찰 전환율, 문서작성 시간 절감률
- Reliability: 파이프라인 성공률, 재시도율, 파싱 실패율

---

## 6) 기술/데이터 의존성

필수 의존성:
- 공고 수집 API 안정성(나라장터)
- 첨부파일 다운로드/파싱(HWP/PDF)
- 회사 프로필 정합성(면허/자본금/실적)
- 평가 근거 저장 및 감사 추적성

리스크:
- 외부 API 변동(스펙/트래픽/인증)
- 문서 포맷 다양성으로 인한 파싱 실패
- 생성 문서 품질 편차

대응:
- API 오류/지연 대비 큐 기반 재시도 + 모니터링
- 파서 다중화 + 실패 샘플 회귀 테스트
- 문서 생성 결과에 근거/검증 체크리스트 강제

## 7) Phase 9+ 확장 원칙

확장 순서:
1. 민간 입찰(LH, 한전, 도시공사 등)
2. HR(채용공고 분석, JD 생성)
3. 법무(계약서 리스크 분석)
4. 회계(입찰 원가 계산)

확장 기준:
- 기존 "수집 → 분석 → 판단 → 문서 생성" 코어 파이프라인 재사용 가능 여부
- 도메인별 규정/근거 데이터 확보 가능 여부
