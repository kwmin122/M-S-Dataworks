# 다중 문서 질의 + @멘션 설계

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 업로드된 N개 문서 중 특정 문서를 지목하여 질문하거나, 전체 문서를 비교 분석하는 기능 구현.

**Architecture:** 프론트엔드 @멘션 자동완성 UI + 백엔드 문서 필터 파라미터 + 한국어 서수 자동감지.

**Tech Stack:** Python (FastAPI, ChromaDB), TypeScript (React 19, Tailwind CSS)

## 핵심 기능

### 1. @멘션 자동완성

ChatInput에서 `@` 타이핑 시 업로드된 문서 목록이 드롭다운으로 표시:

```
사용자: @
┌──────────────────────────────┐
│ 📄 회사소개서.pdf            │ ← 회사 문서
│ 📄 실적증명서.pdf            │
│ 📋 입찰공고_2024-001.pdf    │ ← 분석 문서
│ ──────────────────────────── │
│ 📊 전체 문서 비교            │ ← 특수 옵션
└──────────────────────────────┘
```

- 선택하면 입력창에 `@회사소개서.pdf ` 태그가 삽입됨 (하이라이트 칩)
- 전송 시 태그된 문서명을 `source_files` 파라미터로 백엔드에 전달
- "전체 문서 비교" 선택 시 모든 문서를 개별 검색 후 비교 프롬프트 사용

### 2. 한국어 서수 자동감지 (백엔드)

사용자가 `@` 없이 자연어로 문서를 지칭할 때 백엔드에서 자동 감지:

```
"첫번째 문서에서 요구사항 알려줘" → source_files = [첫번째 업로드 문서]
"두번째 올린 파일의 예산은?"     → source_files = [두번째 업로드 문서]
"마지막 문서"                   → source_files = [가장 최근 문서]
"이 문서들 비교해줘"            → compare mode
```

감지 우선순위:
1. 프론트엔드 `source_files` 명시 전달 → 최우선
2. 백엔드 서수/키워드 regex 감지 → 보조
3. 둘 다 없음 → 전체 문서 검색 (현재와 동일)

### 3. 문서별 검색 + 라벨링

비교 모드일 때 각 문서를 개별 검색하여 LLM에 라벨링된 컨텍스트 전달:

```
=== 문서 1: 회사소개서.pdf ===
[페이지 3] 당사는 ISO 9001 인증을 보유하고 있으며...
[페이지 7] 연간 매출 500억원...

=== 문서 2: 실적증명서.pdf ===
[페이지 1] 2024년 총 15건의 공공사업 수행...
```

## 데이터 흐름

```
[사용자: @회사소개서.pdf 이 문서의 핵심 역량은?]
    ↓
[ChatInput: 태그 파싱 → sourceFiles: ["회사소개서.pdf"]]
    ↓
[POST /api/chat {session_id, message, source_files: ["회사소개서.pdf"]}]
    ↓
[_resolve_doc_scope(): source_files 있으면 그대로 사용]
    ↓
[_build_chat_context(source_files=["회사소개서.pdf"])]
    ↓ engine.search(query, filter_metadata={"source_file": "회사소개서.pdf"})
    ↓
[LLM: 해당 문서 컨텍스트만으로 답변 생성]
    ↓
[Response: {answer, scoped_to: ["회사소개서.pdf"]}]
    ↓
[MessageBubble: "Based on: 회사소개서.pdf" 배지 표시]
```

비교 모드:
```
[사용자: @전체 문서 비교 가장 비용이 큰 사업은?]
    ↓
[source_files: ["*"] (전체)]
    ↓
[문서별 개별 검색 (top_k=6 each) → 라벨링된 컨텍스트]
    ↓
[비교 시스템 프롬프트 + 라벨링 컨텍스트 → LLM 답변]
```

## UI 상세

### @멘션 자동완성 드롭다운

```
┌─────────────────────────────────────────────────┐
│ [ChatInput]                                     │
│ ┌─────────────────────────────────────────────┐ │
│ │ @회사소개                                   │ │
│ └─────────────────────────────────────────────┘ │
│ ┌─ 문서 선택 ──────────────────────────────── │ │
│ │ 🏢 회사소개서.pdf          15 청크          │ │
│ │ 🏢 실적증명서.pdf           8 청크          │ │
│ │ ─────────────────────────────────────────── │ │
│ │ 📋 입찰공고_SW개발.pdf     23 청크          │ │
│ │ ─────────────────────────────────────────── │ │
│ │ 🔄 전체 문서 비교                           │ │
│ └─────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────┘
```

- 🏢 = 회사 문서, 📋 = 분석 문서 (구분 표시)
- 타이핑하면 필터링 (fuzzy match)
- 방향키 + Enter로 선택
- ESC로 닫기

### 태그 칩 (선택 후)

```
┌─────────────────────────────────────────────────┐
│ [@회사소개서.pdf ✕] 이 문서의 핵심 역량은?      │
│                                         [전송]  │
└─────────────────────────────────────────────────┘
```

- 칩은 파란색 배경 라운드 태그
- ✕ 클릭으로 태그 제거
- 여러 문서 태그 가능 (@문서1 @문서2)

### CompanyDocCard "질문하기" 버튼

```
┌─────────────────────────────────────┐
│ 📄 회사소개서.pdf    15 청크  💬  ✕ │
│ 📄 실적증명서.pdf     8 청크  💬  ✕ │
└─────────────────────────────────────┘
```

💬 클릭 → ChatInput에 `@회사소개서.pdf ` 자동 삽입 + 포커스

## 검증

1. @타이핑 → 문서 목록 드롭다운 표시
2. 문서 선택 → 태그 칩 삽입 + 해당 문서만 검색 확인
3. "첫번째 문서에서..." → 서수 감지 → 해당 문서만 검색
4. "전체 문서 비교" → 문서별 개별 검색 → 비교 답변
5. 응답에 "Based on: xxx.pdf" 배지 표시
6. 태그 없이 일반 질문 → 기존과 동일 동작
