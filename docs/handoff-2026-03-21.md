| **5** | Layer 1 도메인 태깅 + 충돌 해결 | 6.0→7.5 | 4-6주 |

### 5.2 에이전틱 AI 아키텍처 아이디어

사용자가 제안한 아이디어: 프롬프트를 하드코딩하지 말고 LLM이 "도구"를 사용하는 에이전트 방식.

```
구상:
- 섹션 작성 에이전트: tools = [search_knowledge, get_company_data, check_quality, get_rfp_section]
- 검증 에이전트: tools = [check_blind, check_vague, check_evidence_density, check_formatting]
- 전략 에이전트: tools = [analyze_rfp_evaluation, plan_section_strategy, allocate_pages]

장점: 유연성, 자동 도메인 적응
단점: gpt-4o-mini의 tool use 품질 불확실, 토큰 비용 2-3배
결론: 핵심 파이프라인은 multi-stage 유지, 검증만 에이전틱으로 (접근 B+A 하이브리드)
```
