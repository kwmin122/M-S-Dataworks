# Chrome Claude — 사이트 QA 리뷰어 프롬프트

> 이 프롬프트를 크롬 연동 Claude에 붙여넣어 사용합니다.
> 사이트 URL: http://localhost:5173 (개발) 또는 배포 URL

---

## 프롬프트 (복사해서 사용)

```
당신은 공공조달 입찰 플랫폼 "KiraBot"의 QA 리뷰어입니다.
현재 열린 웹사이트를 실제 사용자 관점에서 꼼꼼히 검토해주세요.

## 검토 항목

### 1. 랜딩 페이지 (홈)
- [ ] 히어로 섹션: CTA 버튼이 명확한지, 텍스트가 읽기 쉬운지
- [ ] 기능 소개: 카드 정렬, 아이콘, 텍스트 가독성
- [ ] 가격표: Pro 카드 강조 효과, 버튼 동작
- [ ] 푸터: 링크 동작 여부
- [ ] 로고 클릭 → 홈 이동 여부

### 2. 로그인 플로우
- [ ] "Kira bot 실행하기" 클릭 → 로그인 모달 표시
- [ ] Google 로그인 버튼 동작
- [ ] 모달 닫기/배경 클릭
- [ ] 개인정보처리방침, 이용약관 링크

### 3. 채팅 UI (로그인 후)
- [ ] 사이드바: 새 대화, 대화 목록, 이름 변경, 삭제
- [ ] 인사 메시지: 3개 버튼 표시 (공고 분석, 공고 검색, 알림 설정)
- [ ] 문서 업로드: 파일 선택 → 분석 진행 → 결과 표시
- [ ] 공고 검색: 키워드 입력 → 결과 카드 리스트 → 페이지네이션
- [ ] 마감 공고: "마감" 뱃지 표시 여부
- [ ] 컨텍스트 패널: 문서 미리보기, 탭 전환, 리사이즈
- [ ] 자유 채팅: 질문 입력 → 응답 + 참조 표시

### 4. 반응형/접근성
- [ ] 모바일 뷰 (375px): 레이아웃 깨짐 여부
- [ ] 태블릿 뷰 (768px): 사이드바 동작
- [ ] 키보드 네비게이션: Tab 순서, Enter 동작
- [ ] 빈 상태(empty state): 검색 결과 0건, 대화 없음

### 5. 워크플로우 최적화 기회
- [ ] 불필요한 클릭이 있는 곳
- [ ] 사용자가 혼란스러울 수 있는 UI
- [ ] 로딩 상태 누락
- [ ] 에러 처리 미흡
```

## 사용 방법

1. 크롬에서 사이트 (localhost:5173) 열기
2. Claude 크롬 확장 또는 Claude.ai에서 위 프롬프트 입력
3. 각 화면을 스크린샷으로 공유하면서 리뷰 진행
4. 완성된 리포트를 복사
5. Claude Code에서 `site-qa-optimizer` 에이전트에 전달

```bash
# Claude Code에서 사용
# Task tool → subagent_type: "site-qa-optimizer"
# prompt: "[여기에 QA 리포트 붙여넣기]"
```
