# 우리의 목표

> 이 문서는 Kira Bot 프로젝트의 목표와 워크플로우를 **코드 없이** 설명합니다.
> 다른 AI 에이전트나 협업자가 프로젝트를 빠르게 이해하기 위한 브리핑 문서입니다.

## 한 줄 요약

~~"이 공고, 우리가 할 수 있어?" 라는 질문에 30초 안에 답을 얻는 것.~~

**"이 공고, 제안서까지 다 만들어줘" — 입찰의 전체 라이프사이클을 자동화하는 AI BD 팀.**

---

## 1. 우리가 풀고 있는 문제

대한민국 공공조달 시장은 연간 약 160조 원 규모다. 수만 개의 기업이 나라장터(G2B)에서 입찰 공고를 찾고, 수백 페이지의 RFP(제안요청서) 문서를 읽고, 자격요건을 하나하나 대조한다.

현실은 이렇다:

- **하루 수백 건**의 공고가 올라온다. 관심 분야만 추려도 수십 건이다.
- 공고 하나를 제대로 분석하려면 **최소 2~4시간**이 걸린다.
- RFP 문서는 50~300페이지짜리 PDF/HWP이고, 자격요건이 여기저기 흩어져 있다.
- "면허 있어?", "자본금 충분해?", "실적 있어?" — 이 기본적인 확인조차 사람이 수작업으로 한다.
- GO를 결정해도 **제안서 50~200페이지를 처음부터 쓰는 데 2~4주**가 걸린다.
- **제안서 1건 외주 비용**: 300~1,000만원. 중소기업에게는 부담.
- 결과적으로 **좋은 기회를 놓치거나**, **자격이 안 되는 입찰에 시간을 낭비**한다.

**Kira Bot은 공고 발견부터 제안서 제출까지, 입찰의 전 과정을 AI로 자동화한다.**

---

## 2. 우리가 만들고 있는 것

### 제품명: Kira Bot (키라봇)

Kira Bot은 **공공조달 입찰 전 과정 자동화 AI 플랫폼**이다.

핵심 가치 다섯 줄:

1. **자동 수집**: 나라장터 공고를 자동으로 수집하고, 우리 회사와 관련 있는 공고만 골라준다.
2. **즉시 분석**: RFP 문서에서 자격요건을 추출하고, GO/NO-GO를 판단한다.
3. **제안서 자동 생성**: 평가항목에 맞춰 기술제안서, 수행계획서, PPT까지 원클릭으로 생성한다.
4. **학습하는 AI**: 전문가 노하우 + 회사 고유 스타일 + 승패 패턴을 학습하여 점점 좋아진다.
5. **대화형 지원**: 분석 결과와 제안서에 대해 자연어로 질문하고 수정한다.

```
  └── RFP 업로드 → 분석 + 생성 (산업 무관, 수동 업로드)

📋 정부지원사업
  └── 사업계획서 생성 (TIPS/바우처/창업패키지)
```

## 4. 핵심 철학: "먼저 배우고, 그다음 생성한다"

공공기관이 "이거 해줄래?" → 기업이 "내가 할게!" 하고 낙찰받는 구조.
공공기관은 까다롭다. AI가 제안서를 쓰려면, 사람 전문가가 수십 년간 쌓은 노하우를 먼저 학습해야 한다.

### 3계층 학습 모델 — 우리의 차별화

```
┌───────────────────────────────────────────────────────────┐
│  Layer 1: 기본 모델 (Base Model)                            │
│  ─ 유튜브 50~N편 + 블로그 + 법령/공식가이드 학습            │
│  ─ 평가위원 심리, 감점 요소, 제안 나무(Recommendation Tree) │
│  ─ 모든 사용자에게 공통 적용                                 │
│                                                            │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  Layer 2: 회사 맞춤 모델 (Custom Model)               │   │
│  │  ─ "과거 제안서 업로드 → 문체/구조/강점 자동 분석"     │   │
│  │  ─ 회사 CI, 용어사전, 템플릿 학습                      │   │
│  │  ─ RAG 기반 (비용 1/10, 즉시 갱신)                    │   │
│  │                                                      │   │
│  │  ┌───────────────────────────────────────────────┐   │   │
│  │  │  Layer 3: 승패 분석 모델 (Win-Loss Model)       │   │   │
│  │  │  ─ 낙찰 vs 탈락 태그 → "이길 때와 질 때의 차이" │   │   │
│  │  │  ─ 경쟁사 낙찰 제안서 (정보공개 청구 가이드)    │   │   │
│  │  │  ─ 사용할수록 데이터 쌓이는 플라이휠            │   │   │
│  │  └───────────────────────────────────────────────┘   │   │
│  └─────────────────────────────────────────────────────┘   │
└───────────────────────────────────────────────────────────┘
```

**Layer 1** — "제안서의 보편적 법칙"
- 유튜브 강의, 블로그, 공식 가이드, 교육기관 커리큘럼에서 추출한 전문가 노하우
- 평가위원은 캡션과 강조문구만 본다 (평가의 역설)
- 배경-목적-비전의 화자 구분, 제안 나무 구조, 감점 회피 체크리스트
- 모든 고객에게 공통 적용

**Layer 2** — "우리 회사 색깔"
- 과거 제안서 업로드 → 문체, 구조, 강점 표현, CI 자동 분석
- "업계 베스트 프랙티스 + 우리 회사 색깔"의 융합
- 회사가 새 제안서를 쓸 때마다 자동 학습 (RAG — 재학습 불필요)
- **수정 diff 학습**: AI 초안을 사용자가 수정하면 그 차이를 자동 학습 → 다음 생성 시 반영

**Layer 3** — "뭘 했을 때 이기는지"
- 낙찰 vs 탈락 제안서를 비교 분석하여 승패 패턴 추출
- "귀사의 승패 패턴 분석 리포트" — 그 자체로 유료 가치
- 경쟁사 낙찰 제안서(정보공개 청구) → 익명화 인사이트로 모든 고객에게 환원
- 사용할수록 데이터가 쌓이고, 정확해지고, 이탈하기 어려워지는 플라이휠

## 5. 사용자가 경험하는 흐름 (확장 후)

### 시나리오 A: "이 공고로 제안서까지 만들어줘" (핵심 시나리오)

```
사용자 → "소프트웨어 개발 공고 검색"
       → 키라봇이 나라장터에서 실시간 검색
       → 관심 공고 클릭 → 첨부 RFP 자동 다운로드
       → AI가 자격요건 추출 + GO/NO-GO 판정
       → "GO입니다. 제안서 패키지를 만들까요?"
       → 사용자: "만들어줘"
       → 키라봇이 원클릭으로 전체 패키지 생성:
           📄 기술제안서 (DOCX, 100+ pages)
           📋 수행계획서 + WBS (XLSX + DOCX)
           🎯 발표자료 (PPTX, 25장 + 발표 노트)
           📋 실적기술서 / 경력기술서 (DOCX)
           ✅ 제출 체크리스트 + 마감일 알림
       → 사용자가 각 문서를 검토하고 수정
       → "Executive Summary의 수치를 더 추가해줘"
       → 키라봇이 수정된 버전 생성
```

### 시나리오 B: "처음 가입 — 회사 학습"

```
사용자 → 키라봇 가입
       → "과거에 작성한 제안서가 있으시면 업로드해주세요.
          Kira가 귀사의 제안 스타일을 학습합니다."
       → 낙찰 제안서 5건 + 탈락 제안서 3건 업로드 (태그 표시)
       → AI가 분석:
          "귀사의 제안서 스타일:
           - 문체: 격식체, 평균 문장 28자
           - 강점: 인력 경험 강조, 기술 아키텍처 도식 활용
           - 낙찰 패턴: 사례 3건+, WBS 주 단위, 수치 5개+"
       → 이후 모든 제안서 생성에 자동 반영
```

### 시나리오 C: "정부지원사업 사업계획서"

```
사용자 → "AI 바우처 사업 계획서 만들어줘"
       → 키라봇이 AI 바우처 프레임워크 자동 적용
          (AI 기술 활용 계획, 기대효과, 데이터 계획)
       → PSST 구조로 사업계획서 초안 생성
       → 시장 데이터 자동 삽입 (웹서치 기반)
       → 사용자 검토/수정 → 최종 제출본
```

## 6. 시스템이 내부에서 하는 일 (워크플로우)

### 6-1. 공고 수집 파이프라인 (✅ 완성)

```
[나라장터 Open API]
        ↓ 정기 크롤링 (n8n 스케줄러)
[공고 데이터 정제 + DB 저장]
        ↓
[첨부파일 다운로드 + 문서 파싱]
        ↓
[기업 관심설정과 매칭되는 공고 필터링]
        ↓
[자동 평가 작업 생성]
        ↓
[AI 분석 실행 → 결과 저장 → 이메일 알림]
```

### 6-2. 문서 분석 파이프라인 (✅ 완성)

```
[문서 업로드 또는 자동 다운로드]
        ↓
[문서 파싱] — PDF, DOCX, HWP, Excel, PPT
        ↓
[텍스트 청킹 → 벡터 임베딩 → 검색 인덱스 구축]
        ↓
[자격요건 추출] — LLM 멀티패스 + 동의어 사전 17개 카테고리
        ↓
[RFP 요약 생성] — 사업개요 / 핵심요건 / 평가기준
        ↓
[GO/NO-GO 판정] — 결정론적 규칙 우선, 불확실 시 LLM 보조
```

### 6-3. 제안서 생성 파이프라인 (🚀 확장 중)

```
[GO 판정 완료]
        ↓
[proposal_planner] — 제안 전략 수립 에이전트 (Supervisor)
        ↓
    ├── [section_writer × N] — 평가항목별 섹션 병렬 생성
    │       ↓ 3계층 학습 모델 (기본 + 회사 맞춤 + 승패 패턴)
    │       ↓ 회사 역량 DB에서 유사 실적/인력 매칭
    │       ↓ 과거 제안서 라이브러리에서 재사용 가능 콘텐츠 검색
    │
    ├── [wbs_generator] — WBS/일정/간트차트 생성
    ├── [personnel_matcher] — 투입인력 최적 배치
    ├── [cost_calculator] — 원가 산출 (SW기술자 노임단가 기반)
    └── [quality_checker] — 품질 검증 (감점 요소 자동 탐지)
        ↓
[document_assembler] — DOCX/PPTX/XLSX 최종 조립
        ↓
[사용자에게 전달] — 검토/수정 → 최종 제출
```

### 6-4. 학습 파이프라인 (🚀 구축 중)

```
[Layer 1: 기본 모델 구축] — $10 미만으로 전문가 50명분 노하우 구조화

  Step 1: 소스 수집 (3개 채널)
    채널 A: YouTube 50~100편 (자막 추출, 큐레이션 기준: S급 낙찰후기 > A급 평가위원 > B급 컨설턴트)
    채널 B: 공식 문서 30~50편 (조달청 가이드, 법령, 표준 양식 — PDF 직접 수집)
    채널 C: 블로그 100~200편 (trafilatura로 본문 추출, 1000자 미만/광고성 필터링)
    → 총 100만~160만 토큰 원시 텍스트

  Step 2: LLM 2-Pass 정제 (Knowledge Distillation)
    Pass 1: 원문 → 7개 카테고리(structure/evaluation/writing/visual/strategy/compliance/pitfall) 지식 단위 추출
    Pass 2: 중복 제거 + 모순 해소 3단계 파이프라인
      Case 1 (70%): AGREE — 같은 내용, 다른 소스 → 병합 + confidence ↑
      Case 2 (24%): CONDITIONAL — 맥락 다른 규칙 → 둘 다 저장 + condition 필드 (적격심사 vs 기술평가 등)
      Case 3 (6%): CONFLICT — 진짜 충돌 → 가중 투표 (공식문서 우선) + 패배 규칙 deprecated
    → 100만+ 토큰 → 정제된 지식 단위 800~1,800개 (조건부 분기 포함)

  Step 3: 벡터DB 2-Collection 구축
    Collection 1: proposal_knowledge (정제 지식, confidence+freshness 메타데이터)
    Collection 2: proposal_templates (구조화 템플릿, 공종/사업유형별)

  Step 4: Anti-Pattern Gate (능동적 품질 차단)
    pitfall DB → 생성 후 자동 검증 → 감점 사유 0건 보장
    "근거 없는 추상 표현" "블라인드 위반" "표 설명 누락" 자동 탐지+재작성

  Step 5: Golden Test (서비스 투입 전 검증)
    실제 나라장터 공고 10건 → Layer 1 있을 때 vs 없을 때 비교
    목표: 커버리지 30%+ 향상, 치명적 실수 0건

  Quick Start: YouTube 20편 + 공식 가이드 10편 → Mini Layer 1 → 2주 내 A-lite와 동시 투입

  ⚠️ 사람이 해야 하는 작업 (코드가 아닌 큐레이션):
    1. YouTube URL 큐레이션 — 50~100편 선별 + S/A/B/C 등급 부여
    2. 공식 문서 PDF/HWP 수집 — 조달청/과기부/법제처 사이트에서 직접 다운로드
    3. 블로그 URL 큐레이션 — 100~200편 필터링 (광고/스팸 제거)
    4. Golden Test 공고 10건 선정 — IT/건설/용역 등 다양한 업종·규모 포함

[Layer 2: 회사 맞춤 학습]
  과거 제안서 업로드 → 문체/구조/강점 분석 → 회사 프로필 생성

[Layer 2 수정 Diff 학습 루프] — "RLHF for Proposals"
  AI 초안 생성 → 사용자 수정 → diff 추출
      → 1회: 기록만 (학습 안 함)
      → 2회 같은 패턴: 후보로 마킹
      → 3회 이상: Layer 2 자동 반영 + 사용자 알림
  수정률(Edit Rate) = 품질 KPI:
      v1: 수정 45% → v5: 수정 20% → v10: 수정 8% = "학습도 92%"
  → 구독 해지 방어의 핵심 ("92% 학습한 AI를 버릴 건가요?")

[Layer 3: 승패 분석]
  낙찰 vs 탈락 제안서 → 차이점 추출 → 승패 패턴 DB
  경쟁사 낙찰 제안서 → 발주처 선호도 분석
```

### 6-5. 인텔리전스 엔진 (🆕)

```
[낙찰 결과 자동 수집] — 나라장터 낙찰/개찰 결과 API
        ↓
[자동 태그] — 우리 고객의 입찰 건에 낙찰/탈락 자동 표시
        ↓
[Layer 3 자동 업데이트] — 사용자 액션 제로로 승패 패턴 학습
        ↓
[발주처 프로파일링] — 발주처별 선호 패턴, IT 리터러시 수준 분석
        ↓
[GO/NO-GO 2.0] — 자격 충족 + 승률 + 기대수익 종합 판정
        ↓
[입찰 포트폴리오 전략] — "이번 달 이 3개에 집중하세요" 추천
        ↓
[입찰 성장 리포트] — 분기별 승률 추이, 개선점, 다음 분기 전략
```

**GO/NO-GO 2.0 예시**:
```
자격 충족: YES
예상 경쟁률: 8개사
귀사 예상 기술점수: 64/70 (상위 30%)
적정 투찰가: 2.64~2.76억 (추정가의 88~92%)
종합 승률: 약 35%
→ GO (상위권 진입 가능, 기술 차별화에 집중 권장)
```

**입찰 포트폴리오 전략 예시**:
```
이번 달 귀사에 적합한 공고 23건 중:
🏆 Top Pick (승률 40%+): NIA 클라우드 전환, 교육부 LMS 구축
⚡ 도전 가치 (20~40%): 과기부 AI 분석 플랫폼
⚠️ 리스크 (<20%): 국방부 보안시스템
```

**Post-Bid Learning Loop** — 매번 입찰할 때마다 더 똑똑해지는 구조:
- 탈락 시: "왜 떨어졌을까?" AI 자동 추론 → 다음 제안서에 보강
- 낙찰 시: 성공 패턴 강화
- 분기별 성장 리포트: 승률 추이, 기술점수 변화, 개선 권장사항

### 6-6. 민간 입찰 + 정부지원사업 (🆕)

```
공공조달 (코어, 풀 자동화):
  나라장터 자동 수집 → 분석 → 제안서 → PPT → 체크리스트 → 낙찰결과 추적
  = 모든 파이프라인이 자동

민간 입찰 (개방형, 수동 업로드):
  자동차/건설/방산 등 어떤 산업이든
  → "RFP 파일을 업로드하세요" → 분석 + 제안서 생성
  = 엔진은 동일, 수집/추적은 없음

정부지원사업 (사업계획서):
  TIPS/AI바우처/창업패키지 등
  → 사업유형별 프레임워크(PSST 등) 적용 → 사업계획서 자동 생성
```

민간 전략: 수동 업로드로 시장 검증 → 고객 수요 시 자동 수집 확장 (리스크 제로).

## 8. 기술적 철학

### "규칙이 먼저, AI는 나중에"
자본금 비교, 면허 보유 여부 같은 명확한 조건은 규칙으로 판단한다. LLM은 모호한 조건에만 사용. 정확하고, 일관되며, 설명 가능하다.

### "먼저 배우고, 그다음 생성한다"
AI가 제안서를 쓰기 전에, 전문가 50명분의 노하우를 먼저 학습한다. 유튜브, 블로그, 공식 가이드, 교육 커리큘럼에서 "현장 지식"을 추출하여 기본 모델로 만든다.

### "이길 때와 질 때의 차이를 안다"
단순히 "잘 쓴 제안서"를 만드는 게 아니라, "이 회사가 이길 때 뭘 했는지"를 학습하여 승률을 높인다. 낙찰 vs 탈락 비교 분석 → 승패 패턴 → 제안서 생성에 반영.

### "죽어도 복구된다"
시스템이 중간에 꺼져도 데이터가 손상되지 않는다. DB에는 항상 안정된 상태만 기록된다.

### "보안은 기본이다"
모든 요청은 HMAC 서명 검증. 파일 다운로드는 허용된 도메인에서만. 환경변수 하나 빠지면 서버 시작 안 됨.

### "동의어를 안다"
"사업비" = "예산" = "추정가격" = "배정예산". 17개 카테고리 동의어 사전으로 어떤 표현이든 같은 개념으로 인식한다.

### 확장 로드맵 (Phase 3: 준비 + Phase 4: 제출)

**Phase 1: 기반 구축 (2개월) — "A-lite 먼저" 전략**

회사 DB 없이도 RFP 분석 + Layer 1만으로 "범용 제안서 초안" 생성 가능 → **2주 만에 데모**.

| 주차 | 모듈 | 설명 |
|:---:|------|------|
| Week 1-2 | A-lite (제안서 MVP) | Layer 1 + RFP 분석만으로 범용 초안 생성 (60점). 회사 DB 불필요 |
| Week 3-4 | 회사 역량 DB | 실적/인력/과거 제안서 구조화 → 초안 품질 60→85점 |
| Week 5-6 | A-full + 체크리스트 | Layer 2 연동 + 회사 맞춤 제안서 (92점) + 제출 서류 검증 |
| Week 7-8 | 수정 diff 학습 루프 | 사용자 수정 → 패턴 감지 → 자동 학습 → 쓸수록 좋아지는 구조 |
| 병행 | Layer 1 학습 | 유튜브 50편 + 블로그 + 법령 → 노하우 벡터 DB |

**Phase 2: 핵심 자동화 (2~3개월)**

| 모듈 | 설명 |
|------|------|
| 수행계획서 / WBS | 과업분석 → WBS → 간트차트 → 인력배치표 |
| 실적/경력 기술서 | 회사 DB에서 RFP에 맞는 실적/인력 자동 매칭 |
| PPT 발표자료 | 제안서 → 핵심 추출 → PPTX + 발표 노트 + 예상질문 |
| Layer 2+3 학습 | 회사 맞춤 온보딩 + 승패 분석 파이프라인 |

**Phase 3: 고도화 (3~6개월)**

| 모듈 | 설명 |
|------|------|
| 가격제안서 | SW기술자 노임단가 기반 원가 자동 산출 |
| 승률 대시보드 | 입찰 이력 분석, 승률 예측, 공고 추천 |
| 사업계획서 생성기 | 정부지원사업 (TIPS, AI바우처, 창업패키지 등) |
| 민간 입찰 확장 | LH, 한전, 도시공사 크롤러 |

## 12. 경쟁 우위

| 차별점 | Kira Bot | 경쟁사 |
|--------|----------|--------|
| End-to-End | 공고 발견 → 분석 → 제안서 → PPT → 제출 → 학습 | 분석만 또는 생성만 |
| 3계층 학습 | 기본 + 회사 맞춤 + 승패 분석 | 범용 AI만 |
| 인텔리전스 | 발주처 프로파일링 + 승률 예측 + 투찰가 예측 | 없음 |
| Post-Bid Loop | 낙찰결과 자동 추적 → 매번 더 똑똑해짐 | 없음 |
| 네트워크 효과 | 고객 많을수록 인사이트 풍부 (데이터 플랫폼) | 없음 |
| 민간 확장성 | 공공 코어 + 민간 개방형 (엔진 공유) | 공공만 또는 민간만 |
| HWP 지원 | 완성 | 일부/없음 |
| 한국 공공조달 특화 | 조달청 평가기준 내장 | 범용 |
| 나라장터 자동 수집 | 크롤링 완성 | 제한적/없음 |

### 네트워크 효과 — 최종 해자

```
사용자 증가 → 입찰 결과 데이터 축적 → 인사이트 정확도 향상
    → 새 사용자 유입 + 기존 사용자 이탈 방지 → 데이터 더 축적 → (반복)
```

경쟁사가 따라오려면 같은 규모의 사용자 베이스가 필요. 시간으로만 해결됨. 선점자 우위.
