# Kira Bot 사업계획서

**MS Solutions | 2026년 3월**

---

## Executive Summary

Kira Bot은 대한민국 공공조달 입찰의 전 과정(공고 발견 → RFP 분석 → GO/NO-GO 판단 → 제안서·PPT·WBS 자동 생성 → 제출 관리 → 학습)을 자동화하는 AI SaaS 플랫폼이다.

공공조달 시장 225조원, 등록 사업자 50만 개. 그 중 96.7%가 중소기업이다. 이 기업들은 공고 하나를 분석하는 데 2~4시간, 제안서를 쓰는 데 2~4주, 외주 시 300~1,000만원을 쓴다. Kira Bot은 이 과정을 **30초 분석 + 원클릭 제안서 생성**으로 압축한다.

**핵심 차별화**: 분석만 하는 기존 도구(클라이원트, 비드프로)와 달리, Kira Bot은 분석부터 문서 생성까지 End-to-End를 자동화하고, 3계층 학습 엔진(범용 노하우 + 회사 맞춤 + 승패 패턴)으로 쓸수록 좋아지는 구조를 갖췄다.

현재 상태: **Phase 1-2 구현 완료** (제안서 DOCX, PPT, WBS, 실적기술서 자동 생성 + Layer 1 지식 495유닛 탑재 + 수정 diff 학습 루프). 154개 테스트 통과, E2E 검증 완료.

## 1. 문제 정의

### 1.1 시장 배경

대한민국 공공조달 시장은 연간 **225조원** 규모(2024년 기준, 조달청 통계)다. 나라장터(G2B)에 등록된 사업자는 **약 50만 개**, 이 중 96.7%가 중소기업이다.

### 1.2 현재의 고통 (Pain Points)

| 문제 | 현실 | 비용 |
|------|------|------|
| 공고 발견 | 하루 수백 건, 관심 분야만 수십 건 | 담당자 1명 × 2시간/일 |
| RFP 분석 | 50~300페이지 PDF/HWP, 요건 산재 | 2~4시간/건 |
| GO/NO-GO 판단 | 면허·자본금·실적 수작업 대조 | 오판 시 수백만원 낭비 |
| 제안서 작성 | 50~200페이지, 처음부터 집필 | 2~4주, 외주 300~1,000만원 |
| PPT·WBS·실적기술서 | 별도 작성 필요 | 추가 1~2주 |
| 제출 관리 | 서류 누락으로 감점/실격 | 취소 불가 |
| 결과 학습 | 낙찰/탈락 원인 분석 안 함 | 같은 실수 반복 |

**핵심 인사이트**: 중소기업의 BD(사업개발) 팀은 보통 1~3명이다. 한 건의 입찰에 올인하면 다른 기회를 놓친다. 동시에 여러 건을 진행하면 품질이 떨어진다. **시간과 품질의 트레이드오프**가 핵심 고통이다.

### 1.3 기존 솔루션의 한계

| 솔루션 | 하는 일 | 못하는 일 |
|--------|---------|----------|
| 비드프로 | 공고 검색·알림 | 분석 없음, 생성 없음 |
| 클라이원트 | 공고 검색 + AI 분석 | 제안서 생성 없음, 학습 없음 |
| 디마툴즈 | 낙찰가 예측 (ML) | 분석 없음, 생성 없음 |
| 제안서 컨설팅 | 수동 작성 | 건당 300~1,000만원, 2~4주 |
| 범용 AI (ChatGPT 등) | 텍스트 생성 | 공공조달 전문성 없음, HWP 못 읽음 |

**시장 공백**: "분석 + 생성 + 학습"을 하나의 플랫폼에서 제공하는 서비스가 없다.

## 2. 솔루션: Kira Bot

### 2.1 제품 개요

Kira Bot은 **공공조달 입찰 풀 라이프사이클 자동화 AI 플랫폼**이다.

```
공고 발견 → AI 분석 → GO/NO-GO → 제안서 패키지 생성 → 제출 관리 → 결과 학습
   자동        30초       즉시      원클릭 (5종 문서)      체크리스트     자동
```

### 2.2 핵심 기능 (구현 완료)

| 기능 | 설명 | 상태 |
|------|------|------|
| 나라장터 실시간 검색 | 키워드·업무구분·지역·금액 필터 | 운영 중 |
| 문서 자동 파싱 | PDF, DOCX, HWP, HWPX, Excel, PPT | 운영 중 |
| AI 자격요건 추출 | 멀티패스 병렬 + 동의어 사전 17개 카테고리 | 운영 중 |
| RFP 3섹션 요약 | 사업개요·핵심요건·평가기준 | 운영 중 |
| GO/NO-GO 자동 판단 | 규칙 기반 우선 + LLM 보조 (하이브리드) | 운영 중 |
| 제안서 DOCX 자동 생성 | Layer 1+2 지식 기반, 100+ 페이지 | 운영 중 |
| PPT 발표자료 생성 | KRDS 디자인 가이드 + 예상질문 10개 | 운영 중 |
| 수행계획서/WBS 생성 | XLSX 간트차트 + DOCX 보고서 | 운영 중 |
| 실적·경력 기술서 | 회사 DB 기반 자동 매칭 | 운영 중 |
| 제출 체크리스트 | RFP에서 필수 서류 자동 추출 | 운영 중 |
| 맞춤 공고 알림 | 키워드·금액·지역 필터 + 이메일 발송 | 운영 중 |
| 수정 diff 학습 | AI 초안 수정 → 패턴 감지 → 자동 반영 | 운영 중 |
| 대화형 Q&A | RAG 하이브리드 검색 + 참조 페이지 | 운영 중 |
| 프로덕트 투어 | 신규 사용자 온보딩 가이드 | 운영 중 |

### 2.3 3계층 학습 엔진 — 핵심 차별화

```
Layer 1: 범용 지식 (모든 고객 공통)
├── 유튜브 40편 + 블로그 47편 + 공식문서 18편에서 추출
├── 495개 구조화된 지식 유닛 (ChromaDB)
├── 평가위원 심리, 감점 회피, 제안 나무, 배점 전략
└── "전문 컨설턴트 50명분 노하우" — 구축 비용 $10 미만

Layer 2: 회사 맞춤 (고객별 개별)
├── 과거 제안서 업로드 → 문체·구조·강점 자동 분석
├── 사용자 수정 diff → 패턴 감지 (3회 이상 → 자동 반영)
├── RAG 기반 — 재학습 불필요, 즉시 갱신
└── "쓸수록 우리 회사 색깔이 나오는 AI"

Layer 3: 승패 분석 (네트워크 효과) [Phase 3]
├── 낙찰 vs 탈락 비교 → 승패 패턴 추출
├── 발주처별 선호도 프로파일링
├── 사용자 많을수록 데이터 축적 → 정확도 향상
└── "데이터 플라이휠" — 경쟁사 진입 장벽
```

### 2.4 기술 스택

| 계층 | 기술 | 비고 |
|------|------|------|
| 프론트엔드 | React 19 + Vite + TypeScript + Tailwind | 대화형 Chat UI |
| 백엔드 (레거시) | Python FastAPI + ChromaDB | 분석·생성 엔진 |
| 백엔드 (SaaS) | Next.js 16 + Prisma + PostgreSQL | 멀티테넌트 B2B |
| 자동화 | n8n 워크플로우 4종 | 공고 수집·평가·알림 |
| AI 엔진 | OpenAI GPT-4 + 임베딩 + ChromaDB RAG | 하이브리드 검색 |
| 문서 생성 | python-docx, python-pptx, openpyxl, mistune 3.x | DOCX/PPTX/XLSX |
| 보안 | HMAC 서명, CSRF, SSRF 방어, 입력 검증 | 엔터프라이즈 수준 |
| 인프라 | Railway (현재) → AWS/GCP (스케일) | 컨테이너 기반 |

## 3. 시장 분석

### 3.1 시장 규모

**TAM (Total Addressable Market) — 전체 시장**

- 대한민국 공공조달 시장: **225조원/년** (2024, 조달청)
- 등록 사업자: **50만 개**
- IT 서비스·소프트웨어 분야 입찰 참여 기업: **약 8만 개**

**SAM (Serviceable Available Market) — 접근 가능 시장**

- IT/SW 분야 활발한 입찰 기업 (연 5건 이상): **약 2만 개**
- 평균 구독료 월 30만원 가정: **2만 × 360만원 = 720억원/년**

**SOM (Serviceable Obtainable Market) — 획득 가능 시장 (3년)**

- 1차 타겟: IT 서비스 중소기업 (종업원 10~100명)
- 3년 내 목표 점유율 5%: **1,000개사 × 360만원 = 36억원/년**

### 3.2 경쟁 환경

| 경쟁사 | 포지셔닝 | 매출(추정) | 강점 | 약점 |
|--------|---------|-----------|------|------|
| **클라이원트** | AI 공고 분석 | ~5억원(2024) | 가장 유사, AI 분석 | 제안서 생성 없음, 학습 없음 |
| **비드프로** | 공고 검색·알림 | ~30억원 | 대규모 사용자 | 레거시 UI, AI 없음 |
| **디마툴즈** | 낙찰가 예측 | ~10억원 | ML 가격 예측 | 문서 분석/생성 없음 |
| **제안서 컨설팅사** | 수동 작성 | 시장 분산 | 고품질 | 건당 300~1,000만원, 느림 |

**경쟁 포지션 맵**:

```
           생성 능력 (높음)
               ↑
               │  ★ Kira Bot
               │   (분석+생성+학습)
               │
 분석 깊이 ────┼──────────── 범위 (넓음)
 (깊음)        │
               │  ● 클라이원트      ● 비드프로
               │   (분석만)         (검색만)
               │
           생성 능력 (없음)
```

### 3.3 시장 트렌드

1. **공공조달 디지털 전환 가속**: 정부 디지털 플랫폼 정책, 전자입찰 100% 전환 완료
2. **AI 업무 자동화 수용도 증가**: ChatGPT 이후 기업 AI 도구 도입률 급증
3. **중소기업 생산성 도구 수요**: 인력 부족 → SaaS 도구로 대체
4. **정부지원사업 증가**: TIPS, AI 바우처 등 → 사업계획서 수요 증가

- **제안서 외주 비용**: 1건 300~1,000만원 → Kira Bot PRO 월 9.9만원으로 5건 생성

- **시간 절감**: 분석 4시간 → 30초, 제안서 2주 → 1시간 검토
- **기회 비용**: 1건 분석하는 동안 놓치는 다른 공고 → 동시 다건 진행 가능

## 5. Go-to-Market 전략

### 5.1 타겟 고객 세그먼트

**1차 타겟 (Launch)**: IT 서비스 중소기업
- 종업원 10~100명
- 연 5건 이상 공공 입찰 참여
- BD 담당자 1~3명
- **Why**: IT 기업은 AI 도구 수용도가 높고, RFP가 상대적으로 표준화되어 있음

**2차 타겟 (6개월 후)**: 건설·엔지니어링·컨설팅
- 입찰 빈도 높고 제안서 부담 큰 산업
- HWP 문서 의존도 높음 → Kira Bot HWP 파싱 차별화

**3차 타겟 (12개월 후)**: 정부지원사업 신청자
- 스타트업, 소상공인
- TIPS, AI 바우처, 창업패키지 사업계획서 수요

### 5.2 채널 전략

| 채널 | 전략 | 기대 효과 |
|------|------|----------|
| **콘텐츠 마케팅** | "공공입찰 GO/NO-GO 판단법" 블로그·유튜브 | 오가닉 유입, SEO |

Phase 3 (12~24개월): 플랫폼화

## 6. 경쟁 우위 및 해자 (Moat)

### 6.1 단기 해자 (0~12개월)

| 차별점 | 설명 |
|--------|------|
| **End-to-End 자동화** | 유일하게 분석→생성→학습까지 한 플랫폼에서 제공 |
| **HWP 완벽 지원** | 공공조달 핵심 포맷. 경쟁사 대부분 미지원 |
| **3계층 학습** | Layer 1(범용) + Layer 2(회사맞춤) 이미 동작 |
| **한국 공공조달 특화** | 조달청 평가기준·동의어 17개 카테고리 내장 |
| **제안서 생성 품질** | 495유닛 전문 지식 + KRDS 디자인 가이드 적용 |

### 6.2 장기 해자 (12개월+)

| 해자 | 메커니즘 |
|------|---------|
| **수정 학습 lock-in** | 사용자가 수정할수록 AI가 학습 → "92% 학습한 AI를 버릴 건가요?" |
| **데이터 네트워크 효과** | 고객 증가 → 승패 데이터 축적 → 인사이트 정확도 향상 → 더 많은 고객 |
| **회사 DB 축적** | 실적·인력·과거 제안서가 쌓일수록 전환 비용 증가 |
| **시간 선점** | Layer 3 승패 분석은 데이터 축적이 필수 → 후발주자는 따라잡기 어려움 |

## 12. 핵심 지표 (KPI)

| 지표 | 정의 | Y1 목표 |
|------|------|--------|
| MAU | 월간 활성 사용자 | 500 |
| Free→PRO 전환율 | 무료 → 유료 전환 | 15% |
| MRR | 월간 반복 매출 | ₩1,000만원 |
| Churn Rate | 월간 이탈률 | < 5% |
| 수정률 (Edit Rate) | AI 초안 대비 사용자 수정 비율 | 30% → 15% (6개월) |
| NPS | 순추천지수 | > 40 |
| 제안서 생성 건수 | 월간 AI 생성 제안서 수 | 200건 |

**수정률(Edit Rate)**은 Kira Bot의 고유 KPI다.
- v1: 수정 45% → v5: 수정 20% → v10: 수정 8% = "학습도 92%"
- 이 수치가 낮아질수록 고객 고착도가 높아지고, 이탈 방어가 강해진다.
