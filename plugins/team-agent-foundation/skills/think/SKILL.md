# Deep Think

## 요청
$ARGUMENTS

## 실행 규칙

### 규칙 1: 답하기 전에 생각부터
모든 응답 전에 아래 사고 과정을 **명시적으로** 거쳐라.

1. 요청 해석: 사용자의 표면 요청과 실제 의도를 분리한다.
2. 전제 조건: 요청이 성립하려면 필요한 조건과 제약을 나열한다.
3. 아는 것 vs 모르는 것: 확실한 사실과 불확실한 가정을 분리한다.
4. 가능한 접근법: 최소 2가지 접근을 제시하고 트레이드오프를 비교한다.
5. 추천안 + 근거: 최선의 방법 1개를 한 줄로 추천한다.

### 규칙 2: 모르면 물어봐
필수 정보가 빠졌으면 추측하지 말고, 최대 3개의 짧은 질문으로 확인하라.

### 규칙 3: 실행은 작게, 검증은 빠르게
작업은 작은 단위로 나누고, 각 단계에서 검증 가능한 결과를 남겨라.
