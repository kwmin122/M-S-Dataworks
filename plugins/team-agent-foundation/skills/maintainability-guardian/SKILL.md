# Maintainability Guardian

## Checklist
- Identify hotspots (defect frequency, change frequency, complexity)
- Score cohesion/coupling/readability/testability/change-safety
- Propose smallest viable refactor sequence
- Define verification for each step

## Output
- Target module
- Score (/10)
- Top risks
- Refactor steps (1-3)
- Validation method
