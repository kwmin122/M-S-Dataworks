# Title: 제안요청서에 뭐가 들어있을까?
# Category: compliance
# Quality: 중

"제안요청서 받았는데... 뭘 봐야 하죠?"
입찰이나 제안서를 처음 접하신 분들이 가장 많이 하는 질문입니다.
사실, 제안요청서(RFP)는 제안서를 쓰기 위한 '설계도'입니다.
이번 글에서는 정보시스템 구축 사업을 기준으로 제안요청서에 어떤 항목이 들어가는지 정리해 볼게요.
제안요청서(RFP)란?
쉽게 말해,
"이런 시스템을 만들어줄 회사를 찾습니다. 조건은 이렇게 정리했으니 보고 제안서를 써주세요."라는 문서예요.
공공기관이나 기업이 어떤 사업을 하려고 할 때 필요한 조건과 요청사항을 미리 문서로 정리해서 입찰 대상자에게 전달하는 것이죠.
RFP에는 어떤 내용이 들어 있을까요?
정보시스템 구축 사업의 경우, 제안요청서는 보통 아래와 같은 6가지 내용으로 구성됩니다.
| 항목 | 어떤 내용이 들어 있을까요? |
| 사업 개요 | 이 사업을 왜 하는지, 어떤 서비스를 만들지 |
| 현재 상황과 문제점 | 지금 시스템이나 업무의 불편한 점 |
| 사업 추진 방안 | 언제부터 언제까지 어떻게 진행할지 |
| 기술 요청 내용 | 어떤 기능이 필요한지, 어떤 기준을 지켜야 하는지 |
| 제안서 작성법 | 어떻게 써야 하는지, 목차는 어떻게 되는지 |
| 입찰 관련 조건 | 누가 참여할 수 있고, 어떻게 제출해야 하는지 |
하나씩 쉽게 설명드릴게요.
1. 사업 개요
- 이 사업은 왜 필요한가요?
- 어더에 쓰이는 시스템인가요?
- 어떤 효과를 기대하고 있나요?
ex : "민원 처리 시스템을 새로 만들어서 업무 시간을 줄이고, 시민 편의를 높이기 위해 시작합니다."
2. 현황과 문제점
- 지금 어떤 문제가 있어서 개선하려는 걸까요?
- 시스템이 오래됐나요?
- 업무가 너무 복잡한가요?
이 항목은 제안서에서 "왜 이 시스템이 꼭 필요한지" 설득하는 근거가 됩니다.
3. 사업 추진방안
- 이 사업은 몇 개월 동안 진행되나요?
- 어떤 단계로 나눠서 진행되나요?
- 누가 무엇을 담당하나요?
ex : "착수 - 설계 - 개발 - 테스트 - 오픈" 이런 일정이 제시됩니다.
4. 기술 요청 내용
- 어떤 시스템을 만들어야 하나여요?
- 웹으로 만들어야 하나요? 클라우드 기반이어야 하나요?
- 보안, 유지관리, 교육은 어떻게 해야 하나요?
제안서 본문 대부분이 이 항목을 기반으로 구성돼요.
5. 제안서 작성 방법
- 제안서는 몇 쪽으로 써야 하나요?
- 어떤 순서로 구성해야 하나요?
- 어떤 기준으로 점수를 주나요?
ex : "기술 80점, 가격 20점, 총점으로 평가합니다."
6. 입찰 및 제출 조건
- 이 입찰에는 어떤 회사가 참여할 수 있나요?
- 공동수급(컨소시엄)은 가능한가요?
- 언제까지, 어떻게 제출해야 하나요?
마감일, 분량, 제출처 등은 놓치면 탈락할 수 있으니 꼭 체크해야 합니다.
부록도 꼭 챙기세요!
제안요청서에는 보통 '부록'이라는 이름으로 기술 명세서나 상세 요구사항이 따로 붙어 있어요.
예를 들면,
- 시스템 요구 정의서
- 기능 명세서
- 기술적용계획표
- 각종 서식
이 문서들은 제안서에 구체적인 근거로 활용됩니다.
한 줄 요약
제안요청서(RFP)는 제안서를 어떻게 써야 하는지 알려주는 안내서입니다.
| 구성 항목 | 체크 포인트 |
| 사업 개요 | 왜 하는 사업인가요? |
| 현황과 문제 | 어떤 문제를 해결해야하나요? |
| 추진방안 | 일정과 역할은 어떻게 되나요? |
| 기술요청 | 어떤 시스템이 필요한가요? |
| 작성법 | 어떻게 써야 하나요? |
| 입찰조건 | 누가, 언제까지, 어떻게 제출하나요? |
