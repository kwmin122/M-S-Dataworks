# Title: 제안요청서 분석, 누가 어떤 일을 해야 할까요?
# Category: structure
# Quality: 상

제안서 잘 쓰려면 '역할 나누기'가 먼저입니다.
"제안서는 어떻게 나눠서 쓰는 건가요?"
입찰 제안서를 처음 준비할 때,
많은 분들이 이런 고민을 하세요.
사실 제안서 작성은 혼자서 하기에는 너무 많은 일입니다.
그래서 팀원들과 역할을 나누는 게 무엇보다 중요해요.
이 글에서는 제안요청서(RFP)를 분설할 때 각 담당자가 맡아야 할 일을 쉽게 설명해 드릴게요.
제안서는 팀워크로 씁니다.
좋은 제안서는 혼자 만드는 게 아닙니다.
PM(책임자), 기술담당, 영업담당, 전략기획자 등 각자의 역할이 잘 나뉘어야 완성도가 높아져요.
☞ 주요 역할과 업무 정리
아래는 실제 제안서 준비할 때 자주 등장하는 역할과 그들이 맡아야 할 일입니다.
1. PM(프로젝트 책임자)
주요 역할
- 제안서 전체 방향을 정해요
- 어떤 항목이 중요하고 점수를 많이 받을 수 있는지 분석해요
- 전체 일정과 팀을 조율해요
쉽게 말하면,
제안서를 이끄는 '리더'역할이에요. 방향키를 잡아주는 사람!
2. 사업관리 담당
주요 역할
- 회사 소개, 실적 정리
- 제안서에 들어갈 조직표, 일정표 등 행정적인 부분 정리
- 체크리스트를 만들어 누락되는 게 없게 도와줘요
쉽게 말하면,
제안서의 '틀'을 잡고, 빠지는 게 없게 관리하는 사람!
3. 예산 담당자(보통 PM 또는 영업이 함께함)
주요 역할
- 견적서 작성
- 제품 가격 정리
- 경쟁사의 가격 흐름 파악
쉽게 말하면,
'얼마에 낼 것인가'를 결정하고 정리해 주는 사람!
4. 기술 담당자(PL)
주요 역할
- 제안요청서 안의 기술 요구사항을 꼼꼼히 읽고
- 어떤 기능이 필요한지, 어떤 시스템이 맞는지 정리
- 실제로 어떻게 만들지 계획까지 세워요
쉽게 말하면,
'이걸 어떻게 만들지'를 고민하고 쓰는 사람!
시스템, 하드웨어, 소프트웨어, 운영, 서비스 등 분야별로 담당을 나누면 더 좋아요.
5. 전략 담당자
주요 역할
- 제안서를 읽는 사람이 빠르게 이해하고 "이 회사가 잘하겠구나!"라는 인상을 받을 수 있도록 스토리를 짜고 메시지를 구성해요.
쉽게 말하면,
제안사를 설득력 있게 쓰도록 도와주는 이야기 설계자!
6. 영업/자료 수집 담당
주요 역할
- 필요한 자료를 모으고
- 고객, 협력사와 소통하고
- 제출 기한 맞춰 제안서를 정리해서 제출해요
쉽게 말하면,
뒤에서 꼼꼼히 챙기고 밖에서도 소통도 하는 실무자 역할!
실무에서 생기는 문제들
역할을 나누지 않으면 이런 문제가 생겨요.
- 누가 뭘 맡았는지 몰라서 중복 작업
- 중요한 항목이 빠져서 감점
- 마감 직적에 정신없는 수정 지옥
그래서 처음에 역할표를 만드는 게 필수예요!
역할 정리표 예시
| 이름 | 맡은 역할 | 해야 할 일 |
| 김책임 | PM | 방향 잡기, 일정 관리 |
| 이기술 | 기술 PL | 기능 정리, 해결책 작성 |
| 박전략 | 전략 담당 | 메시지 구성 |
| 최영업 | 자료 수집, 견적 정리 | 미시작 |
☞ 엑셀, 노션, 구글시트 어디서든 쉽게 만들 수 있어요!
요약정리
☞ 제안서 분석 = 팀플에이
각자 역할이 정리되면 일도 빠르고 실수도 줄어들어요.
| 구분 | 핵심 역할 |
| PM | 방향 설정, 일정 잡기 |
| 사업관리 | 실적∙조건표∙일정 정리 |
| 예산 | 견적서, 가격 정리 |
| 기술 | 기능 파악, 기술 해결책 작성 |
| 전략 | 메시지 구성, 스토리 설계 |
| 영업 | 자료 수집, 제출 관리 |
제안서는 '글쓰기'도 중요하지만, '협업'이 먼저입니다.
잘 쓰려면, 먼저 잘 나눠보세요.
