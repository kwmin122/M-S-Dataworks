# Title: 제안요청서 평가 기준, 어떤 기준으로 점수를 줄까요
# Category: evaluation
# Quality: 상

제안서에 점수는 어떻게 매겨질까요?
"입찰 제안서 쓰신 써야 하는데... 무엇을 기준으로 평가받는 걸까?"
제안서를 잘 쓰기 위해선 어떻게 점수를 받는지부터 아는 게 중요합니다.
이걸 모르고 그냥 쓰면, 정작 중요한 부분에 점수를 못 받을 수 있어요.
이번 글에서는 SW개발, ISP(정보화 전략계획), 운영유지보수 등
사업 유형에 따라 어떻게 평가 기준이 달라지는지 쉽고 간단하게 알려드릴게요.
제안서 평가 기준이란?
입찰에서는 가격 외에도 기술적으로 얼마나 좋은 제안인가?를 따져 봅니다.
이걸 기술성 평가라고 해요.
다섯 가지 항목을 기준으로 평가자가 점수를 줍니다.
1. 개발 계획
2. 개발 내용
3. 관리 계획
4. 지원 방안
5. 협력사 참여
이 다섯 가지를 제안서에 잘 담아야 높은 점수를 받을 수 있어요.
사업별 평가 항목과 점수는 어떻게 다를까요?
사업의 성격에 따라 어떤 항목에 점수를 많이 주는지 달라집니다.
| 평가 항목 | SW 개발 | ISP | 유지보수 |
| 개발 계획(이해도/전략) | 30점 | 35점 | 20점 |
| 개발 내용(기능/방법) | 35점 | 35점 | 20점 |
| 관리 계획(일정/품질) | 10점 | 15점 | 20점 |
| 지원 방안(교육/보안) | 5점 | 5점 | 25점 |
| 협력사 참여 | 20점 | 10점 | 10점 |
| 총합 | 100점 | 100점 | 100점 |
사업별로 어디에 집중해야 할까?
SW 개발 사업
∙ 개발 계획 + 내용이 65점
☞ 기술력, 시스템 구현 능력 강조!
ISP 사업(정보화 전략계획)
∙ 이해도와 추진 전략이 핵심
☞ 논리적이고 설득력 있는 구성 필요
운영유지보수 사업
∙ 운영 안정성, 지원 능력에 무게
☞ 유지보수, 장애 대응, 교육 등 강조
제안서 쓸 때, 이렇게 대응해 보세요!
점수 많은 항목에 더 많은 내용과 분량을 써야 해요.
예를 들면,
- 유지보수 제안서에는 '지원 방안'을 자세히!
- ISP 제안서는 '이해도'와 '전략'을 중심으로!
- 개발 사업 제안서는 기능과 성능 설명을 구체적으로!
평가 기준과 제안서 목차를 같이 맞춰서 구성하면, 평가자가 찾기 쉬워서 점수를 더 잘 줄 수 있어요.
제안서 평가 항목 정리표
| 항목 | 제안서에 넣을 내용 예시 |
| 개발 계획 | 유사 프로젝트 경험, 이해도, 전략 |
| 개발 내용 | 기능 설명, 개발 방식, 화면 예시 |
| 관리 계획 | 일정표, 품질보증 계획 |
| 지원 방안 | 유지보수 조직도, 교육계획 |
| 협력사 참여 | 중소기업과 협력 내용, 역할 분담 |
한 줄 정리
제안서를 잘 쓰는 비결?
"평가표를 먼저 보는 것"
점수 많이 주는 항목에 더 힘을 쏟으세요.
전략적으로 작성하면 평가자가 잘 볼 수밖에 없어요.
