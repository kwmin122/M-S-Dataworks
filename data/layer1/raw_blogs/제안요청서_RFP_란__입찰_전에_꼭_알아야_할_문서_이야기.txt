# Title: 제안요청서(RFP)란? 입찰 전에 꼭 알아야 할 문서 이야기
# Category: compliance
# Quality: 중

"입찰 제안서를 써야 합니다."라는 말, 많이 들으시죠?
그럴 때 가장 먼저 해야 할 일은
제안요청서, 즉 RFP(Request For Proposal)를 제대로 이해하는 것입니다.
제안요청서, 쉽게 말해 뭐예요?
제안요청서는(RFP)는 이렇게 이해하면 됩니다.
"이런 일을 해줄 회사를 찾습니다. 조건은 이러하니, 제안서를 써서 제출해 주세요."
즉, 발주기관이 사업을 시작하기 전에 입찰 대상자에게 알려주는 공식 요청 문서입니다.
제안요청서는 어떤 내용이 들어 있나요?
RFP에는 사업에 필요한 거의 모든 정보가 담겨 있어요
- 어떤 사업인지
- 어떤 기능이나 시스템이 필요한지
- 예산은 어느 정도인지
- 사업 기간은 얼마나 걸리는지
- 제안서는 어떤 형식으로, 언제까지 제출해야 하는지
- 평가 기준은 무엇인지.
이 내용을 보고 우리는 제안서의 방향과 내용을 결정하게 됩니다.
왜 이렇게 중요한가요?
RFP는 단순한 공문서가 아닙니다.
제안서를 잘 쓰기 위한 설계도 같은 역할을 해요.
| 이유 | 설명 |
| 방향을 잡을 수 있어요 | 무엇을 써야 할지 정확히 알 수 있어요 |
| 고객의 요청을 파악할 수 있어요 | 고객이 원하는 걸 빠짐없이 담을 수 있어요 |
| 평가 기준을 미리 알 수 있어요 | 어떤 항목이 중요한지 파악 가능 |
| 경쟁사보다 더 좋은 전략을 세울 수 있어요 | 분석이 곧 경쟁력입니다. |
실무에서 꼭 확인하는 항목 5가지
RFP를 받을 때 아래 내용을 꼭 체크하세요.
- 이 사업은 왜 시작되었을까?
- 고객이 요구하는 기능/서비스는 무엇일까?
- 예산은 얼마고, 일정은 어떻게 될까?
- 제안서는 어떤 형식으로, 언제까지 제출해야 할까?
- 평가 기준은 어떻게 되어 있나?
이 내용을 체크리스트나 표로 정리해 두면 팀원들과 공유하기도 좋아요.
예를 들어 정리하면
| 구분 | 예시 설명 |
| 사업 개요 | "온라인 민원 시스템 고도화" |
| 요구사항 | "반응형 웹, 연계 API, 관리자 페이지 포함" |
| 예산 & 일정 | "2억 원/계약일로부터 6개월" |
| 제출 조건 | "A4 100쪽 이내, 제안서 2부, 제출 마감 06.30까지" |
| 평가 기준 | "기술 80점, 가격 20점, 제안서 심사 포함" |
이런 내용을 꼼꼼히 확인하고 정리하면 제안서 쓰는 게 훨씬 쉬워집니다.
한 줄 정리
제안요청서(RFP)는 "고객이 우리에게 요구하는 조건을 공식적으로 설명한 문서"입니다.
