# 코드 리뷰 스킬

PR이나 코드 변경 사항에 대해 리뷰가 필요할 때 활성화됩니다.

## 리뷰 프로세스

### 1단계: 변경 범위 파악
```bash
git diff --stat
git diff --name-only
git log --oneline -5
```

### 2단계: 필수 체크리스트
- [ ] **버그/로직 오류**: 의도와 다르게 동작할 수 있는 코드
- [ ] **보안 취약점**: SQL injection, XSS, 인증 우회, 민감 데이터 노출
- [ ] **에러 처리 누락**: 예외가 적절히 처리되는가
- [ ] **엣지 케이스**: null, 빈 값, 경계값, 동시성 이슈
- [ ] **타입 안정성**: type hints가 정확한가

### 3단계: 권장 체크리스트
- [ ] **코드 가독성**: 변수명, 함수명이 명확한가
- [ ] **테스트 커버리지**: 새 코드에 테스트가 있는가
- [ ] **성능**: N+1 쿼리, 불필요한 반복, 메모리 누수
- [ ] **CLAUDE.md 규칙 준수**: 프로젝트 규칙을 따르는가
- [ ] **문서화**: 공개 API에 docstring이 있는가

### 4단계: 리뷰 결과 출력
```
## 리뷰 결과

### 심각도: 높음 (반드시 수정)
- [파일:라인] 설명

### 심각도: 중간 (수정 권장)
- [파일:라인] 설명

### 심각도: 낮음 (참고)
- [파일:라인] 설명

### 잘한 점
- 긍정적인 피드백
```

## 원칙
- 사소한 스타일 이슈는 무시 (린터가 처리)
- 구체적인 개선안을 함께 제시
- 비판이 아닌 제안 톤으로 작성
- 잘한 부분도 언급
