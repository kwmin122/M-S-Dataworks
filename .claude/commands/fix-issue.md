GitHub 이슈 또는 버그를 분석하고 수정해줘.

이슈: $ARGUMENTS

## 수정 프로세스

### 1. 이슈 분석
- 이슈 내용 파악 (GitHub 이슈면 `gh issue view` 사용)
- 재현 조건 확인
- 영향 범위 파악

### 2. 원인 탐색
- 관련 코드 검색 및 분석
- 에러 로그 확인 (있는 경우)
- 근본 원인 (root cause) 식별

### 3. 수정 계획
- Plan Mode로 수정 계획 수립
- 최소 변경 원칙 적용
- 사이드 이펙트 검토

### 4. 구현
- 수정 코드 작성
- 관련 테스트 추가 (같은 버그가 재발하지 않도록)

### 5. 검증
- 기존 테스트 전체 통과 확인
- 새로 추가한 테스트 통과 확인
- 수동 테스트 (필요시)

### 6. 문서화
- 커밋 메시지에 이슈 번호 포함 (`fix: ... (#이슈번호)`)
- CLAUDE.md의 MISTAKES LOG에 관련 내용 추가
- PR 생성 (필요시)

수정 전에 원인 분석 결과를 보여주고 확인을 받아줘.
