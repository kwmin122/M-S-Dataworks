방금 변경된 파일들을 검토하고 더 단순하게 리팩토링해줘.

## 대상 파일 확인
```bash
git diff --name-only HEAD~1
```

$ARGUMENTS

## 단순화 기준

### 코드 구조
- 불필요한 추상화 제거 (3회 미만 반복은 추상화 불필요)
- 중복 코드 통합
- 함수는 하나의 역할만 수행
- 중첩 깊이 2단계 이내로 유지
- 조건문 단순화 (early return 활용)

### 가독성
- 더 명확한 변수/함수명 사용
- 매직 넘버 → 상수로 추출
- 복잡한 조건식 → 설명적 변수로 분리
- 불필요한 주석 제거 (코드가 스스로 설명하도록)

### 성능
- 불필요한 반복 제거
- N+1 쿼리 패턴 확인
- 불필요한 데이터 로딩 제거

## 규칙
- 동작 변경 없이 구조만 개선 (리팩토링 원칙)
- 변경 후 반드시 기존 테스트 통과 확인
- 린트 체크 통과 확인
- 변경 전후 비교 diff를 보여줘
